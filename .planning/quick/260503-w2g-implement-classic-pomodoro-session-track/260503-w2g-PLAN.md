    ```typescript

      setIsRunning(false);
      setCompletedFocuses(0);
      setRemainingSec(durationForMode);
      saveTimer(buildState({ isRunning: false, remainingSec: durationForMode, completedFocuses: 0 }));
    };
    ```

    This ensures that clicking Reset not only stops and rewinds the timer, but also ends the current "session" — the completed-focuses counter goes back to zero, so the long-break detection (every 4 completedFocuses) starts fresh after reset.

    **Rationale:** A "session" in the user's mental model = the period between resets. When the user resets, they intend to start a fresh session. The completedFocuses counter drives long-break scheduling, so it should match the current session's pomodoro count.
